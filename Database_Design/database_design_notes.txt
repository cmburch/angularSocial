
*********************DATABASE DESIGN NOTES*********************

* Data Modeling
-  data modeling - is the process of creating a model for the data you want to store
-  4 step understand the business requirements, build a conceptual model,
    build logical model, and build physical model

* Conceptual Model
- conceptual model represent the entities and their relationship

* Normalization
- 1st normal form each cell should have a single value 
  and don't have repeated columns
- 2nd normal form every table should descibe one entity and every column in that
  table should describe that entity
- 3rd normal form a column in a table should not be derived from other columns
